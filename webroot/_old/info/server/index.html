WebCall Server


Telecom in a Box

A single WebCall server can host up to 2M simultaneous callees* and serve up to 40M calls per day.

*Callees are long-term connected users, awaiting incoming call. Callers dialing in are supported in addition to the given number of callees.


WebCall Cluster

WebCall Server is fully clusterable. Eight boxes (6x WebCall + LB + DB) are able to host up to 12M simultaneous callees and serve up to 240M calls per 24h.

NGINX* can be used for load balancing HTTPS. WebCall server takes care of load balancing WSS.



Distributed WebCall

WebCall users can be hosted in multiple locations. Users will be able to call each other from and to any location:



Whether such locations are hosted by the same party, or by separate organisations, doesn't really matter. The total number of callee slots can simply be added up. So for example, if each of these locations supports 12M callees, the total number of callee slots available will be 36M. Users will be able to call each other, no matter where they are connected to.
